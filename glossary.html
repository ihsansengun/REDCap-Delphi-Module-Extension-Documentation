1. Delphi Method Terms

TermDefinition
Delphi MethodA structured research technique where a panel of experts answer questionnaires over multiple rounds. After each round, they see a summary of the group's responses and can revise their answers. The goal is to converge toward group agreement.
Delphi RoundOne cycle of the survey. In Round 1, participants give their initial ratings. In Round 2 (and beyond), they see feedback from the previous round and can change their answers.
Panel / PanellistThe group of experts participating in the Delphi study. Each individual expert is a panellist.
Controlled FeedbackThe summary information shown to participants in later rounds. It typically includes group averages and/or distributions, plus the participant's own previous answer. "Controlled" means the researchers decide what feedback to show.
AnonymityA core Delphi principle: participants do not know who else is on the panel or who gave which answer. This prevents dominant individuals from influencing the group.
IterationThe process of repeating rounds. Each round gives participants a chance to reconsider their views in light of the group's responses.
Stakeholder GroupA category of participant based on their role or background (e.g. Patients, Clinicians, Nurses). Feedback is often broken down by stakeholder group so participants can see how different perspectives compare.
ConsensusWhen the group reaches sufficient agreement on an item. What counts as "sufficient" depends on the consensus rule being used.
Dissensus / DisagreementWhen the group has NOT reached agreement. Responses are too spread out to meet the consensus threshold.
Core Outcome Set (COS)A standardised minimum set of outcomes that should be measured and reported in all clinical trials for a specific condition. Delphi surveys are commonly used to develop these.

2. Organisations, Tools & Platforms

TermDefinition
REDCapResearch Electronic Data Capture. A free, secure web application for building and managing online surveys and databases for research studies. Used by 7,950+ institutions in 163 countries.
External Module (EM)A plug-in for REDCap that adds custom functionality without modifying REDCap's core code. The Delphi module is an External Module.
Action TagA special instruction written in a REDCap field's annotation box (e.g. @DELPHI={...}). It tells the External Module how to behave for that specific question.
COMET InitiativeCore Outcome Measures in Effectiveness Trials. An international initiative that brings together people interested in developing and applying core outcome sets. Hosts DelphiManager. URL: comet-initiative.org
DelphiManagerA commercial online tool by the COMET Initiative specifically designed for running Delphi surveys for core outcome set development. Paid service.
eDelphiAnother commercial software platform for conducting Delphi surveys online. Paid service.
ExpertLensA system developed by RAND for gathering opinions from large groups of experts. Paid service.
Surveylet / CalibrumA commercial Delphi survey platform supporting multiple consensus measurement methods. The consensus metrics reference document describes their approach.
OCTRUOxford Clinical Trials Research Unit. The research unit at the University of Oxford that developed the original REDCap Delphi Module.
EChartsAn open-source JavaScript charting library. Already bundled with the module (version 5.5.6) for rendering bar charts.
NIHR BRCNational Institute for Health and Care Research, Biomedical Research Centre. Funds the ongoing development of this module at Oxford.
RANDThe US research organisation that originally developed the Delphi method in the 1950s for military forecasting.

3. Central Tendency (Averages)

TermDefinition
MeanThe arithmetic average. Add up all responses and divide by the number of respondents. Example: scores of 3, 5, 7, 8, 7 → mean = 30 ÷ 5 = 6.0. Best for interval/ratio data that is not skewed.
MedianThe middle value when all responses are sorted from lowest to highest. Example: 3, 5, 7, 7, 8 → median = 7. If even count, it's the average of the two middle values. Best for ordinal data or skewed distributions.
ModeThe most frequently chosen value. Example: 3, 5, 7, 7, 8 → mode = 7 (appears twice). Best for nominal/categorical data.
Consensus ScoreThe computed central tendency value (mean, median, or mode depending on the rule selected) that represents the group's collective answer.
Which average to use? Nominal data → Mode. Ordinal data → Median. Interval/ratio (not skewed) → Mean. Interval/ratio (skewed) → Median.

4. Spread & Dispersion

TermDefinition
Standard Deviation (SD)A measure of how spread out the responses are from the mean. Low SD = responses cluster tightly. High SD = widely scattered. If mean = 7 and SD = 0.5, most people scored between 6.5 and 7.5.
VarianceThe square of the standard deviation. Used in calculations but less intuitive than SD for interpretation.
IQR (Interquartile Range)The range of the middle 50% of responses. Calculated as Q3 minus Q1. A small IQR means the middle half of respondents are tightly clustered. Example: IQR = 1 means the middle 50% scored within 1 point of each other.
Q1 (First Quartile)The value below which 25% of responses fall. Also called the 25th percentile.
Q3 (Third Quartile)The value below which 75% of responses fall. Also called the 75th percentile.
PercentileThe value below which a given percentage of data falls. The median is the 50th percentile.
DistributionThe pattern of how responses are spread across all possible values. Shown as a frequency count and often displayed as a bar chart.
RangeThe difference between the highest and lowest response values. If scores go from 2 to 9, the range is 7.

5. Consensus Rules

RuleDefinition
IQR ≤ 1 & SD ≤ 1.5Consensus is reached if the Interquartile Range is 1 or less AND the Standard Deviation is 1.5 or less. This means the group's responses are tightly clustered. Simple yes/no outcome.
Range ConsensusThe rating scale is divided into equal ranges. Consensus is reached if a sufficient percentage (typically 70%) of responses fall within any single range.
Range Consensus 2Scale divided into 2 equal ranges (Low/High). Consensus if 70%+ are in either range.
Range Consensus 3Scale divided into 3 ranges (Low/Medium/High). Consensus if 70%+ are in any one range. Most commonly used in COS/Delphi studies.
Range Consensus 4Scale divided into 4 ranges (Very Low / Low / High / Very High).
Range Consensus 5Scale divided into 5 ranges (Very Low / Low / Neutral / High / Very High).
High-Only VariantA variation where only the highest range counts. Consensus is only reached if 70%+ are in the "High" or "Very High" range. Used to identify items with strong positive agreement.
70/15 RuleConsensus IN if 70%+ score in the High range (e.g. 7–9 on a 1–9 scale) AND fewer than 15% score in the Low range (1–3). Commonly used in COS Delphi studies.
70/15 BidirectionalExtension: also defines consensus OUT when fewer than 15% score High AND 70%+ score Low. Items meeting neither condition are UNDETERMINED.
MajorityConsensus based on vote counting. Absolute majority: one choice gets > 50%. Simple majority: one choice gets more votes than any other (plurality).
Consensus ThresholdThe percentage of agreement required for consensus. Typically 70% but configurable. If set to 75%, then 75% of responses must fall within the target range.

6. Consensus Status

StatusMeaningBadge
IN / Agreement / ConsentThe item has reached consensus. The group sufficiently agrees, according to the selected rule and threshold.IN
OUT / Disagreement / DissentThe item has NOT reached consensus. Responses are too spread out or do not meet the threshold.OUT
UNDETERMINED / UndecidedThe result is borderline or the rule produces a third state (e.g. bidirectional rules where the item is neither clearly agreed nor clearly disagreed upon).UNDETERMINED

7. Stability & Strength Metrics

TermDefinition
Group StabilityA measure of how strong the consensus is. Even if consensus is reached, stability tells you whether it's a strong or weak agreement. Expressed as a percentage: higher = stronger.
Coefficient of Variation (CV)Standard deviation divided by the mean, as a percentage. Measures spread relative to the average. Used for rating/ordinal questions. Group Stability = 100% − CV.
CV over Range (CVR)A variant of CV for numeric entry questions. Standard deviation divided by the full possible range (max − min). Used when the mean could be zero or negative.
Percentage StabilityFor categorical questions: the percentage of respondents who chose the consensus answer. If 80% chose the same option, percentage stability is 80%.
Stability ThresholdThe minimum group stability required for consensus to be confirmed. If threshold is 50% and stability is only 35%, consensus is NOT confirmed.
Round-to-Round StabilityComparing responses between rounds to see if opinions are converging or diverging. Not yet implemented but the architecture supports it.

Worked Example

7 respondents rate an item on a 1–5 scale with scores: 1, 1, 3, 3, 5, 1, 2

StepValue
Mean16 ÷ 7 = 2.29
Standard Deviation1.50
CV1.50 ÷ 2.29 = 65.5%
Group Stability100% − 65.5% = 34.5%
Threshold50%
Result34.5% < 50% → Consensus NOT confirmed

8. REDCap & Technical Terms

TermDefinition
Instrument / FormA questionnaire or data collection form within a REDCap project. Each Delphi round is typically a separate instrument.
EventIn REDCap's longitudinal design, an event represents a time point or study phase. Round 1 and Round 2 are typically separate events.
RecordA single participant's data in REDCap. Each panellist has one record containing their responses across all rounds.
FieldA single question or data point within an instrument.
Radio ButtonAn input where the respondent selects exactly one option. The Delphi module requires rating questions to use radio buttons.
Matrix QuestionA grid layout where multiple related questions share the same answer options. Displayed as rows with shared column headers.
Likert ScaleA rating scale where respondents indicate their level of agreement or importance. Common scales: 1–5, 1–7, 1–9, 0–10.
PipingREDCap's syntax for referencing data from other fields or events, written as [event_name][field_name].
@DELPHI TagThe action tag used to configure the Delphi module for a specific question. Contains JSON specifying display type, data source, groups, and calculation settings.
preRoundA setting in the @DELPHI tag that tells the module where to find the participant's Round 1 answer. Uses REDCap piping syntax.
calcA new setting: "manual" (default, current behaviour) or "auto" (compute statistics automatically).
HookA point in REDCap's page rendering where an External Module can inject custom code.
Control CenterREDCap's administration area where system-wide settings are managed.
PHPThe programming language REDCap and its External Modules are written in.
JavaScript (JS)The programming language that runs in the web browser. Used to render charts and manipulate the survey page.
JSONJavaScript Object Notation. A lightweight data format for configuration and data exchange. Example: {"type": "single", "calc": "auto"}
CacheA temporary store for computed results so the module doesn't recalculate on every page load.
